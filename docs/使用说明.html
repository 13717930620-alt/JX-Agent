🚀 快速开始

1

安装 Node.js

确保已安装 Node.js 18+。访问 https://nodejs.org/ 下载 LTS 版本。

2

双击运行安装程序

运行「安装程序.bat」，按提示选择安装目录、输入 API Key、选择模型。

3

启动 HyperAgent

安装完成后可以选择立即启动，或以后双击桌面快捷方式启动。

4

开始使用

CLI 模式直接输入指令对话。网页版在浏览器访问 http://localhost:3000，开始与 HyperAgent 对话。

🔐 核心原则：明确指令 = 授权

当您给出明确的操作指令时，HyperAgent 在该任务范围内直接执行，无需额外授权。提问 vs 指令由内置的反任务幻觉系统区分，无需记忆额外的"咒语"。

✅ 直接下指令即可:
用户: "帮我创建一个文件"
 → HyperAgent: 直接创建文件

✅ 提问不会触发执行:
用户: "如何创建文件？"
 → HyperAgent: 回答如何创建文件（不执行操作）
💡 权限级别:
• 信息（只读查询）：查时间、系统信息 — 默认
• 控制（文件/命令/进程）：创建文件、运行命令 — 明确指令后自动授权
• 管理（系统配置）：注册表、服务管理
• 危险（高风险操作）：格式化、关机 — 硬编码拦截

🎯 核心能力详解

💬 智能对话

HyperAgent 能理解复杂的对话意图，自动追踪对话话题，记住用户偏好。

用户: 帮我看看电脑的 CPU 使用率
 → 意图识别: 查询 → 执行 sys_info → 返回结果

用户: 如何用 Python 读取 Excel 文件？
 → 改写检测: 这是提问，不是命令 → 给出代码示例

用户: 写个脚本把桌面上所有 .jpg 文件重命名
 → 任务分解: 分解为 ①列出文件 ②遍历重命名 ③验证结果

🖥️ GUI 桌面控制（UI-TARS）

通过截图理解屏幕内容，自动执行鼠标/键盘操作。不需要知道坐标或元素选择器。

用户: 打开 Chrome 浏览器，访问百度搜索"天气"
 → [截图 → 找到Chrome图标 → 双击 → 等待启动 →
    截图 → 找到地址栏 → 点击 → 输入文字 → ...]

用户: 帮我截图看看桌面上有什么
 → gui_analyze_screen("描述桌面内容")
 → "桌面上有 3 个文件夹，1 个 Chrome 快捷方式..."

🧠 记忆增强

HyperAgent 会记住跨会话的关键信息，包括用户偏好、重要决定、项目上下文。

会话1: "我喜欢用 VS Code 写代码"
会话2: (重启后) "帮我创建一个 Python 项目"
       → 记得用户偏好 VS Code → 创建项目并用 VS Code 打开

🔧 任务自动分解

复杂任务会自动拆解为子任务，带检查点和进度追踪。

用户: 整理桌面文件，把图片放到一个文件夹，文档放到另一个

 → 自动分解:
   ① 创建 "图片" 和 "文档" 文件夹
   ② 搜索桌面上的 .jpg/.png 文件 → 移动到 "图片"
   ③ 搜索桌面上的 .doc/.pdf 文件 → 移动到 "文档"
   ④ 验证结果并报告

 → 进度: "已完成 2/4 步..."

🤖 多智能体协作

遇到复杂任务时，5个专业智能体角色可以协作完成。

协调者: 分解任务 → 分配研究员和程序员
研究员: 搜索最新 AI 新闻 → 整理关键信息
程序员: 生成数据可视化图表
审查员: 检查报告质量和准确性
协调者: 汇总最终报告给用户

✨ v5.0 新功能详解

1. 代码差异对比 (Diff/Apply)

生成和运用 unified diff 格式补丁，自动计算增删行和变更统计。

# 对比两段代码的差异
帮我对比以下两段代码的差异:
原文: "function add(a,b){return a+b}"
改后: "function add(a,b){return a + b;}"

2. LSP 代码智能

连接语言服务器，获取专业的代码诊断和分析。需要先安装对应的 LSP 服务器：

# JavaScript/TypeScript
npm install -g typescript-language-server typescript

# Python
pip install pyright

# Go
go install golang.org/x/tools/gopls@latest
# 启动后可以用:
帮我检查 index.js 文件有没有语法错误
查看 index.js 第5行第10列的类型信息

3. AST 代码结构分析

无需外部依赖，直接解析 JavaScript/TypeScript 代码结构。

# 分析代码复杂度
帮我分析以下代码的圈复杂度:
function process(data) {
  if (data.valid) {
    for (const item of data.items) {
      if (item.active) { processItem(item); }
    }
  }
}

4. 真实语义搜索 (Embedding)

支持三种 Embedding 模式，大幅提升记忆检索精度：

模式配置精度依赖
APIEMBEDDING_MODE=api★★★★★需 API Key
LocalEMBEDDING_MODE=local + ollama★★★★需 ollama
Builtin默认（零配置）★★★无

5. 检查点恢复（长时间任务）

长时间、多步骤的任务会自动保存执行状态，中断后可恢复。

# 创建一个多步骤任务（会自动在后台保存检查点）
帮我做三件事: 先查看系统信息，再创建一个测试文件，最后列出桌面文件

# 中断后重启，再次输入相同任务
# HyperAgent 会自动检测到之前的检查点，从中断处继续执行

6. 工作记录 (Work Record)

HyperAgent 会自动记录每次对话的工作目标、进度和关键发现，中断后快速恢复。

# 查看工作记录
.work

# 中断后恢复
第一次使用: 创建文件 test.txt → 系统自动保存工作记录
  ↓ (意外关闭 / Ctrl+C / 崩溃)
第二次启动: HyperAgent 读取上次工作记录 → 自动注入恢复上下文
            → 知道"刚才在创建文件" → 可以直接继续

⚡ 工作模式

1. 交互模式（CLI）

node HyperAgent_Main.js
命令功能
.status查看系统状态（所有组件运行情况）
.stats查看详细统计信息
.devices查看已注册的设备列表
.learn查看持续学习报告
.work查看工作记录和恢复状态
.help显示帮助
.quit / .exit退出程序

2. 服务模式（Web + API）

node HyperAgent_Main.js server

提供 REST API：

  • POST /api/chat — 发送任务
  • POST /api/chat/stream — SSE 流式对话
  • POST /api/chat/tools — 工具调用模式
  • GET /api/status — 系统状态
  • GET /api/stats — 统计信息
  • GET /api/work-record — 工作记录

3. 远程访问

在对话中说"我要出去了"或"开启远程访问"，HyperAgent 会生成公网链接。到外面后用浏览器访问即可继续对话。

⚠️ 远程访问时安全提示
建议设置 HYPERAGENT_AUTH_TOKEN 防止未授权访问。

⚙️ 环境变量配置

在安装目录的 .env 文件中可以修改配置。复制 .env.example 为 .env 后编辑。

# LLM 主适配器 — 可选: deepseek / glm / minimax / qwen
LLM_ADAPTER=deepseek

# API 密钥（至少配置一个）
DEEPSEEK_API_KEY=sk-xxxxx

# 多模型路由
MULTI_MODEL_ENABLED=true

# Embedding 模式 (v5.0) — api / local / builtin / hybrid
EMBEDDING_MODE=hybrid

# 记忆功能
VECTOR_STORE_ENABLED=true
CROSS_SESSION_MEMORY=true
KG_ENABLED=true

# GUI 控制 (UI-TARS)
UI_TARS_ENABLED=true
UI_TARS_ALLOW_KEYBOARD=false

# 多智能体
MULTI_AGENT_ENABLED=true

# 任务检查点 (v5.0)
CHECKPOINT_ENABLED=true

# Web 服务端口
PORT=3000

🔧 故障排除

⚠️ 启动报 "No API key set"
确保 .env 文件已正确配置 API Key。
⚠️ Puppeteer 启动失败
运行 npx puppeteer browsers install chrome 手动安装 Chromium。
⚠️ 端口被占用
服务模式会自动尝试递增端口，查看控制台输出的实际端口号，或修改 .env 中的 PORT。
⚠️ Embedding 未生效
启动时观察控制台输出：[MemoryPipeline] 初始化完成 (XX项索引, embedding=api) 表示 API 模式正常。builtin 表示使用内置 TF-IDF 模式。
⚠️ 指令没有被执行
1. 尽量使用"帮我做X"、"创建X"等明确措辞
2. 检查 API Key 和 URL 是否正确
3. 查看控制台输出中的错误信息
💡 获取帮助
• 在 CLI 中输入 .help 查看可用命令
• 查看 .env 文件中的详细配置说明
• 运行 .status 查看系统组件状态
