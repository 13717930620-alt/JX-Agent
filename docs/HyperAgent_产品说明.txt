【核心理念】

让任何承载体装上 HyperAgent 之后，就变成一个有自己的智能体，
使用者用简单的语言就能指挥它实现目标。这就是智能化。

        承载体 + HyperAgent = 有头脑的智能体

传统的设备需要人去学习它的操作方法——复杂的控制面板、专用
的指令语言、繁琐的操作手册。而 HyperAgent 的模式完全相反：

  · 设备装上 HyperAgent，就有了感知、理解、决策、执行的能力
  · 使用者只需要说出目标，不需要学习设备的操作方式
  · HyperAgent 负责理解意图、判断可行性、执行操作、反馈结果
  · 实现了从"人适应机器"到"机器适应人"的根本转变

这不是一个工具，而是一个让设备拥有智能的操作系统。


【v5.0 最重要的升级——代码工具链 + 真实 Embedding】

HyperAgent v5.0 在 v4.0 内置模型的基础上，新增两大核心能力：

一、代码工具链

  • Diff/Apply：生成和运用 unified diff 格式补丁，支持代码审查
  • LSP 集成：连接语言服务器，获取诊断/补全/跳转/类型信息
  • AST 分析：无需外部依赖，直接解析代码结构、计算圈复杂度

二、真实 Embedding 语义搜索

  • 支持 text-embedding-3 (OpenAI)、BGE、jina-embeddings
  • 内置 TF-IDF + N-gram 引擎作为零配置备选
  • 语义搜索精度大幅提升，记忆检索更准确

三、检查点恢复 + 工作记录

  • 长时间任务自动保存执行上下文，中断后可恢复
  • 自动记录工作目标/进度，重启后恢复上下文


【v4.0 核心——内置模型，成为承载体真正的"大脑"】

HyperAgent 内置模型不依赖任何外部程序，完全用纯 JavaScript 实现，
真正实现了"零外部依赖，开箱即用"。

这个大脑有三个与生俱来的特征：


【特征一：完全内在——它就在承载体里，哪里也不去】

HyperAgent 的内置模型不是调用远程 API，不是连接本地服务进程，
而是作为承载体操作系统的一部分，直接运行在承载体内部。

  · 不需要联网 —— 所有的推理、分析、决策都在本地完成
  · 不需要外部 API Key —— 不需要任何第三方服务的密钥
  · 不需要安装任何外部程序 —— 不需要 Ollama、Python、Conda、
    CUDA、或任何运行时环境。只需要 Node.js
  · 开机即用 —— 装好 HyperAgent，内置模型就自动就绪

这意味着：HyperAgent 内置模型是承载体的"原生大脑"，
而不是一个需要外部供电的"外挂设备"。它和承载体是一体的。

内置模型的能力包括：

  文本嵌入      character n-gram TF，384 维向量
  自动摘要      TF-IDF 句子评分 + 位置偏置
  文本分类      8 个预置类别库 + 关键词匹配评分
  实体提取      10 种正则模式 + 上下文消歧
  文本分析      多维度统计（情感/复杂度/关键词/趋势）
  差异比较      行级 diff + 数值均值变化检测
  智能聊天      7 类意图模式匹配 + 模板生成


【特征二：自我进化——它每天都在成长，越用越聪明】

HyperAgent 内置模型不是一个静态的程序，而是一个持续进化的
认知系统。它有三层进化机制同时运行：

第一层：三环持续学习（ContinualLearner）

每时每刻都在后台运行，三条环路并行：

  吸收环（每 60 秒）：
    采集 CPU、内存、磁盘、进程等系统状态
    调用本地模型分析状态变化，提取关键信息
    存入 L1 记忆层

  分析环（每 5 分钟）：
    从 L2 记忆中检测重复出现的模式
    挖掘实体之间的关联
    分析系统状态的变化趋势

  进化环（每 15 分钟）：
    将高频模式升华为"核心洞察"存入 L3
    根据积累的洞察校准行为策略
    生成进化报告，记录成长轨迹

第二层：四级自进化引擎（SelfEvolver）

按经验量自动升级进化深度：

  L1 微进化 — 每条新经验：实时更新统计权重
  L2 小进化 — 50 条经验或 5 分钟：模式检测 + 策略学习
  L3 中进化 — 500 条经验或 1 小时：概念重构 + 知识图谱扩展 + 自我评估
  L4 大进化 — 5000 条经验或 24 小时：全面认知重构 + 进化阶段升级

第三层：认知框架（CognitiveFramework）

内置的推理系统不依赖任何外部 LLM：

  演绎推理    if-then 规则匹配（内置 6 条 + 持续学习）
  类比推理    Jaccard 关键词相似度 + 类型匹配
  因果推理    时序相关性分析
  归纳推理    统计泛化 + 趋势检测

同时，所有外部 LLM 的调用结果也会被吸收进记忆系统——
外部 API 是可选的"营养补充"，而不是大脑本身。

三层进化机制同时运行的结果是：
  · 使用第 1 天和第 100 天的 HyperAgent，智能水平完全不同
  · 它记住用户的行为模式、偏好习惯、常见的任务类型
  · 同样的任务，越执行越高效——因为它从不重复犯错
  · 它积累的是"属于这个承载体"的知识，而不是通用知识


【特征三：吸收全部信息——它是承载体的"全息记忆"】

HyperAgent 内置模型吸收的信息范围，远远超过传统智能体：

  系统层 —— 持续采集
    CPU 使用率、内存占用、磁盘空间、进程列表、
    网络连接、系统运行时间、服务状态、注册表变化

  用户层 —— 每一次对话
    用户的问题、指令、反馈、偏好、习惯、
    沟通风格、常用的工具链、偏好的结果格式

  执行层 —— 每一次操作
    执行了哪些命令、读写哪些文件、访问哪些网站、
    哪些操作成功、哪些失败、失败的原因是什么

  时间层 —— 全量记录
    所有信息按时间线组织：L0（7天原始日志）→
    L1（30天关键提炼）→ L2（90天结构化知识）→
    L3（永久核心洞察）。高频使用的信息自动晋升，
    低频信息自动过期。记忆系统是有生命的。

  图谱层 —— 知识关联
    A.U.D.N. 整合器用 LLM 判断每条信息是存是删。
    双时态知识图谱记录每个事实的有效时间窗口。
    层次化 RAG 让检索先找摘要再下钻到细节。

这意味着：如果你把 HyperAgent 装在一台电脑上并持续使用
一个月，它对这台电脑的了解程度可能超过任何人类管理员。
它知道电脑的哪些配置容易出问题、用户喜欢怎么组织文件、
什么时间段系统负载最高、哪个程序经常崩溃——因为它一直
在"吸收"，一直在"学习"。


【这三个特征加在一起意味着什么？】

一台刚装好 HyperAgent 的电脑，和一个用了三个月的电脑，
区别是——后者已经"认识"这台电脑了。

它认识电脑的性能规律：什么时间段负载高，什么程序吃内存。
它认识用户的工作习惯：喜欢什么格式的报告，常用的工具链。
它见过成百上千次的执行成败，知道哪些操作容易出错。
它把这一切沉淀为 L3 核心洞察——这些洞察构成了它对这台
承载体的"理解"。

而这一切，不需要联网，不需要训练，不需要标注数据，
不需要 AI 专家。只需把 HyperAgent 装上，它自己就会
从零开始，一点一点地"长出"对这个承载体的理解。

这不是一个工具，这是承载体"生而自带"的认知系统。


【v5.0 新增功能详解】

代码差异对比 (Diff/Apply)

  HyperAgent v5.0 内置 unified diff 生成和应用工具：
  · 自动计算增删行和变更统计
  · 输出标准的 unified diff 格式
  · 支持补丁应用

LSP 代码智能

  连接语言服务器，获取专业的代码诊断和分析：
  · 代码诊断（语法错误检查）
  · 代码补全建议
  · 类型信息查询
  · 符号定义跳转
  支持 JavaScript/TypeScript、Python、Go、Rust、Java 等语言。

AST 代码结构分析

  无需外部依赖，直接解析 JavaScript/TypeScript 代码：
  · 提取函数和类定义
  · 计算圈复杂度
  · 分析调用关系

真实语义搜索 (Embedding)

  三种 Embedding 模式可选：
  · API 模式：使用 OpenAI text-embedding-3 或 jina-embeddings
  · Local 模式：通过 ollama 运行 BGE 模型
  · Builtin 模式：内置 TF-IDF，零配置，始终可用

检查点恢复

  长时间任务自动保存执行状态：
  · 每完成一个子任务自动保存检查点
  · 工具循环执行中每 5 步自动保存
  · 任务中断后重新运行可自动恢复

工作记录

  自动记录每次对话的工作目标、进度和关键发现：
  · 每次对话后自动更新
  · 退出时强制保存
  · 重启后自动注入恢复上下文


【设计哲学】

1. 明确指令即授权

使用者不需要额外的"授权咒语"。当你说"帮我创建一个文件"时，
HyperAgent 直接执行——因为明确的操作指令本身就是授权。
提问不会触发执行，内置反任务幻觉系统自动区分。

2. 大脑内生，不假外求

HyperAgent 的内置模型是承载体的"原生大脑"，不是外部 AI
的代理。它不需要联网、不需要 API Key、不需要 Ollama、
不需要 Python——装好就能自主思考。外部 LLM 是可选的
"营养补充"，而不是必需品。

3. 承载体抽象化

所有设备都抽象为统一的承载体接口：

   DeviceAbstraction → DeviceManager → ToolExecutor

每一种设备只需实现这个接口，定义自己的能力和安全边界，
就能被 HyperAgent 统一调度。

4. 安全第一

智能体拥有控制权，但必须在安全边界内运行。五级安全体系：
  · Level 1 — 信息查询：无需额外授权（默认）
  · Level 2 — 常规控制：明确指令即授权
  · Level 3 — 高风险操作：双重确认
  · Level 4 — 管理员操作：需手动降级安全级别
  · Level X — 绝对禁止：硬编码底线（格式化/关机等）

【安全体系】

1. 承载体安全边界

每个承载体定义自己的安全参数范围：禁止指令列表等。
超出范围的指令直接被拒绝。

2. 分级授权

  查询类（读文件、看状态）—— 默认允许
  控制类（写文件、执行命令）—— 明确指令即授权
  高风险类（写注册表、电源管理）—— 需确认
  禁止类（格式化、删除系统文件）—— 代码层面硬限制

3. 危险命令过滤

Shell 执行层内置危险命令检测，自动拦截不可逆操作。

4. 反任务幻觉系统

防幻觉系统确保每个工具调用走完完整生命周期，
从根本上消除了 LLM 伪造工具结果的可能性。
